# Kintara Bots — авто-фарм ресурсов

Авто-кликеры для браузерной изометрической игры **Kintara** (воксель, вид
сверху-сбоку). Компьютерное зрение (OpenCV) находит ресурс, кликает мышью
(WinAPI), ждёт добычу по подсветке (глоу), сам панорамирует карту когда цели
кончились.

Каждый ресурс — отдельный бот в своей папке. Готовы **дерево** и **камень+уголь**;
у каждого есть версия **x4** (4 аккаунта в квадрантах 2×2 одного монитора). Все
боты умеют **сам перезаходить на сервер** при вылете/очереди (см. ниже).

## Структура репозитория

```
kintara/

│   ├── win_input.py     # WinAPI клик/драг/скролл, DPI-aware, мультимонитор
│   ├── reconnect.py     # ЗРЕНИЕ перезахода: ловит UI-попапы → действие
│   └── ui/              # PNG-шаблоны UI перезахода (matchTemplate)
│       ├── ok.png  error.png  wait.png  server_list.png
│       └── server_2.png  server_3.png  server_4.png
│
├── tree/                # бот рубки деревьев (готов)
│   ├── bot.py             главный цикл: захват → детект → клик → ждать → пан
│   ├── tree_detector.py   ядро CV: картинка → список деревьев + сигнал глоу
│   ├── debug_detect.py    тест детекции БЕЗ кликов (рисует рамки)
│   ├── diag_glow.py       диагностика подсветки рубки во времени
│   ├── test_drag.py       тест перетягивания карты (before/after)
│   └── assets/            эталонные кадры для тестов
│       ├── screen.png
│       └── highlight.png

│   ├── bot.py             захват → детект → клик → ждать добычу → пан
│   ├── stone_detector.py  CV: картинка → список камней/угля
│   ├── debug_detect.py    тест детекции без кликов
│   ├── diag_capture.py    замер сигнала добычи (заслон телом)
│   └── images-examples/   эталонные кадры (камень + уголь)
```

`common/` — общий для всех ботов: `win_input.py` (клик/драг/скролл),
`reconnect.py` (зрение перезахода) и `ui/` (шаблоны попапов). Скрипты ботов
подключают их через `sys.path` (бутстрап в начале файла), путь считается от
`__file__`, так что запуск работает из любой папки.

## Перезаход на сервер (все боты)

Игра вылетает / кидает в очередь / показывает меню выбора сервера. Боты ловят
это и сами возвращаются в мир — высший приоритет, выше фарма.

- `common/reconnect.py` — **только зрение**: берёт ЦВЕТНОЙ кадр, читает **текст
  кнопок через OCR (RapidOCR)**, возвращает действие. Масштаб/разрешение/монитор-
  независимо, шаблоны `common/ui/*.png` больше не нужны. Без захвата экрана и без
  мыши (это делает бот через `win_input`).
- Приоритет проверок (`emergency_ui_check`): очередь (`QUEUE` → ждать, ничего не
  жать) → `OK` (вылет → клик) → `RETRY` (ошибка → клик) → `Play Now` (главное
  меню → клик) → `Back` (не тот сервер → клик) → `SERVER N` (нужный виден → клик)
  → `Select a server` (список открыт, сервера не видно → скролл вниз искать).
- Каждый аккаунт знает **свой** сервер (`TARGET_SERVER` в одиночных ботах;
  `QUADRANT_*_SERVER` в x4). Координаты действий — локальные внутри региона,
  бот добавляет offset монитора/квадранта.
- Шаблоны в `common/ui/` сняты на **чужом разрешении**; мульти-скейл терпит
  небольшую разницу, при сильном расхождении — **переснять под свой экран**
  (вырезать попапы из своего скриншота, те же имена файлов).

## Режим x4 (4 аккаунта)

`tree-4x/bot.py` и `stone-coal-4x/bot.py` гоняют 4 окна игры «плиткой» 2×2 на
одном мониторе (каждый квадрант = свой логин/сервер).

- **Курсор в системе один.** 4 бота не двигают его разом. Но бот почти всё
  время **ждёт** (перс идёт + добывает). Пока акк-0 ждёт — курсор свободен,
  кликаем акк-1/2/3. ~4× за счёт **интерливинга пауз**, не второго курсора.
- Каждый аккаунт = **конечный автомат** (SCAN → WAIT_APPEAR → WAIT_VANISH → …,
  либо PAN). Планировщик по кругу даёт каждому **один тик**: ≤1 захват региона
  + ≤1 действие мышью. Блокирующего ожидания нет → аккаунты не держат друг друга.
  Все дедлайны по `time.time()`, не по итерациям.
- Регионы — авто 2×2 квадранты `MONITOR_INDEX` (`build_regions`). Включить/
  выключить аккаунты — флаги `QUADRANT_TL/TR/BL/BR`. Сервер на квадрант —
  `QUADRANT_*_SERVER`.
- ⚠️ Квадрант = ¼ экрана → объекты мельче. Пороги площади (`MIN_CLICK_*`)
  калибруй на debug-кадре квадранта.

## Установка

```bash
pip install -r requirements.txt
```

> `keyboard` на Windows иногда требует терминал **от админа** (глобальная 'q'
> для стопа). Не срабатывает стоп — запусти от администратора.

## Запуск (дерево)

Все команды из папки `tree/`:

```bash
cd tree
```

### 1. Проверь детекцию на своём экране (без кликов)

```bash
python debug_detect.py --screen --monitor 1
```

Через 2 сек снимет экран. Смотри `debug_result.png` (пишется в `tree/`):
зелёная рамка + точка = дерево (точка = куда кликнет), красная = отклонено.
Маски `debug_green_mask.png` / `debug_brown_mask.png` — крутить HSV в
`tree_detector.py`. На эталон-кадре: `--image assets/screen.png`.

### 2. Сухой прогон кликера

В `bot.py` поставь `DRY_RUN = True`, затем:

```bash
python bot.py
```

Печатает выбор целей и паузы, мышь НЕ трогает. Проверь логику.

### 3. Боевой режим

`DRY_RUN = False` в `bot.py`, выстави свой `MONITOR_INDEX`. Запусти, переключись
в игру за 3 сек. Стоп — клавиша **q**.

### Диагностика

- `python test_drag.py` — проверить что карта тянется (before/after + % изменения).
- `python diag_glow.py` — кликает дерево, ~20с логирует глоу (калибровка рубки).

## Калибровка

Все крутилки — в `CONFIG` сверху `bot.py` и `tree_detector.py`. Подробно с
обоснованием каждого порога — в `CLAUDE.md` (§9 список крутилок, §10 история
решений). HSV/пороги калибровались на `2560×1600`; на другом разрешении/яркости
проверяй на своём захвате.

## Статус

- **tree** — работает: детект, клик (мультимонитор+DPI), ожидание рубки по глоу
  с учётом дистанции, чёрный список ложных, обход карты змейкой, стоп по 'q'.
- **stone-coal** — работает: детект камня+угля, клик, сигнал добычи (заслон
  телом), обход карты. См. `stone-coal/README.md`.
- **tree-4x / stone-coal-4x** — работают: 4 аккаунта в квадрантах 2×2,
  интерливинг через единый курсор, перезаход на сервер per-quadrant.
- **перезаход на сервер** — во всех ботах (одиночные + x4): ловит вылет/очередь/
  меню сервера, сам возвращается в мир.
