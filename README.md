# Kintara Bots — авто-фарм ресурсов

Авто-кликеры для браузерной изометрической игры **Kintara** (воксель, вид
сверху-сбоку). Компьютерное зрение (OpenCV) находит ресурс, кликает мышью
(WinAPI), ждёт добычу по подсветке (глоу), сам панорамирует карту когда цели
кончились.

Каждый ресурс — отдельный бот в своей папке. Сейчас готов **дерево**, на очереди
**камень**.

## Структура репозитория

```
kintara/
├── common/              # общий код для всех ботов
│   └── win_input.py     # WinAPI клик/драг, DPI-aware, мультимонитор
│
├── tree/                # бот рубки деревьев (готов)
│   ├── bot.py             главный цикл: захват → детект → клик → ждать → пан
│   ├── tree_detector.py   ядро CV: картинка → список деревьев + сигнал глоу
│   ├── debug_detect.py    тест детекции БЕЗ кликов (рисует рамки)
│   ├── diag_glow.py       диагностика подсветки рубки во времени
│   ├── test_drag.py       тест перетягивания карты (before/after)
│   └── assets/            эталонные кадры для тестов
│       ├── screen.png
│       └── highlight.png
│
├── stone-coal/          # бот добычи камня И угля (в работе)
│   ├── bot.py             захват → детект → клик → ждать добычу → пан
│   ├── stone_detector.py  CV: картинка → список камней/угля
│   ├── debug_detect.py    тест детекции без кликов
│   ├── diag_capture.py    замер сигнала добычи (заслон телом)
│   └── images-examples/   эталонные кадры (камень + уголь)
│
├── requirements.txt     # общие зависимости
├── README.md            # этот файл
└── CLAUDE.md            # полный контекст проекта (логика, история решений)
```

`common/win_input.py` общий — оба бота кликают/тянут карту одинаково. Скрипты
в `tree/` подключают его через `sys.path` (бутстрап в начале файла), путь
считается от `__file__`, так что запуск работает из любой папки.

## Установка

```bash
pip install -r requirements.txt
```

> `keyboard` на Windows иногда требует терминал **от админа** (глобальная 'q'
> для стопа). Не срабатывает стоп — запусти от администратора.

## Запуск (дерево)

Все команды из папки `tree/`:

```bash
cd tree
```

### 1. Проверь детекцию на своём экране (без кликов)

```bash
python debug_detect.py --screen --monitor 1
```

Через 2 сек снимет экран. Смотри `debug_result.png` (пишется в `tree/`):
зелёная рамка + точка = дерево (точка = куда кликнет), красная = отклонено.
Маски `debug_green_mask.png` / `debug_brown_mask.png` — крутить HSV в
`tree_detector.py`. На эталон-кадре: `--image assets/screen.png`.

### 2. Сухой прогон кликера

В `bot.py` поставь `DRY_RUN = True`, затем:

```bash
python bot.py
```

Печатает выбор целей и паузы, мышь НЕ трогает. Проверь логику.

### 3. Боевой режим

`DRY_RUN = False` в `bot.py`, выстави свой `MONITOR_INDEX`. Запусти, переключись
в игру за 3 сек. Стоп — клавиша **q**.

### Диагностика

- `python test_drag.py` — проверить что карта тянется (before/after + % изменения).
- `python diag_glow.py` — кликает дерево, ~20с логирует глоу (калибровка рубки).

## Калибровка

Все крутилки — в `CONFIG` сверху `bot.py` и `tree_detector.py`. Подробно с
обоснованием каждого порога — в `CLAUDE.md` (§9 список крутилок, §10 история
решений). HSV/пороги калибровались на `2560×1600`; на другом разрешении/яркости
проверяй на своём захвате.

## Статус

- **tree** — работает: детект, клик (мультимонитор+DPI), ожидание рубки по глоу
  с учётом дистанции, чёрный список ложных, обход карты змейкой, стоп по 'q'.
- **stone-coal** — в работе: детект камня+угля готов, скелет бота + сигнал
  добычи (заслон телом) готовы. См. `stone-coal/README.md`.
