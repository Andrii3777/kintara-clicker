# Stone+Coal Bot — авто-добыча камня и угля (в работе)

Бот добычи КАМНЯ и УГЛЯ в Kintara. Оба ресурса добываются одинаково → ловим
одним детектором (уголь = тот же тёплый низко-S цвет, просто ТЕМНЕЕ). Та же
механика что у дерева. Детектор + скелет бота готовы.

## Что готово

- `stone_detector.py` — ядро CV: картинка → список камней. Без кликов/захвата.
- `debug_detect.py` — тест детекции БЕЗ кликов на примерах/экране (сохраняет
  и сырой кадр `dbg_*_input.png` — тюнить на нём же без перезахвата).
- `bot.py` — главный цикл: захват → детект → клик → ждать добычу по глоу → пан
  змейкой. Скелет из `tree/bot.py`, общий `common/win_input`. Стоп — 'q'.
- `diag_glow.py` — кликает камень, 20с логирует глоу в точке (замер RISE/FALL).
- `images-examples/` — эталонные кадры: камень (1-stone, 2-stones, lots,
  between-trees, full-map), уголь (coal-1/2/3). На них калибровались пороги.

## Как находим камень и уголь (замеры по примерам)

Оба — СЕРЫЙ/тёмный воксель-блоб на траве. Сигнатура HSV (замер реальных пикселей):
- камень: **H 20-26, S 34-51, V 72-110** (медиана V~88)
- уголь:  **H 12-28, S 35-66, V 42-101** (медиана V~60, ТЕМНЕЕ)

Общее: тёплый H, **НИЗКАЯ S** (трава S>125, ствол дерева S>150 → оба выпадают),
V 40-150. Диапазон в `CONFIG`: `STONE_LOWER/UPPER`. Нижний `V=40` включает
тёмный уголь (раньше был 72 и уголь отсекался — теперь он целевой).

Фильтры контура: площадь `MIN_STONE_AREA=150` (режет крапины на стыках тайлов и
краях брусчатки фонтана; калибровано на полном экране 2560×1600 — на сильно
уменьшенных кропах мелкие камни <150px выпадают, но в живой игре они крупнее) /
`MAX_STONE_AREA=7500` (замер: дома/строения ~9400-9700, песчаный холм ~6300 →
порог между ними; дома выпадают, холм проходит). Плотность `MIN_EXTENT`,
вытянутость `MAX_ASPECT` (отсекает тропинки/края крыш). Мягкая проверка
`GRASS_RING_MIN` — рядом должна быть трава (камни в природе). UI-зоны
(`IGNORE_ZONES`) — хотбар/миникарта/иконки.

`MORPH_CLOSE iters=1` (не 2): 2 слипал соседние камни через траву в рваный
блоб → терялись кластеры.

**Фильтр текстуры** (`STONE_VSTD_MIN`/`STONE_SSTD_MIN`): затенённые боковые
грани зелёных воксель-блоков по ЦВЕТУ = камень (H/S/V совпадают), но ОДНОРОДНЫЕ.
Настоящий камень/уголь текстурный. Выкид если разброс яркости И насыщенности ОБА
низкие (Vstd<9.5 И Sstd<5.5). Тёмный уголь: Vstd низкий, но Sstd высокий (чёрные
+серые куски) → проходит. Замер: ложь (8/4, 5/1), уголь-1 (7/8), камень (Vstd>10).

## Запуск (из папки stone-coal/)

```bash
cd stone-coal
python debug_detect.py --all                                  # все примеры
python debug_detect.py --image images-examples/lots-of-stones.png
python debug_detect.py --screen --monitor 1                   # свой экран
```

Смотри `dbg_<имя>_result.png`: зелёная рамка = камень (точка = клик), красная =
отклонено (size/shape/ui/no_grass). Маски `*_stone_mask.png` / `*_grass_mask.png`
— крутить HSV в `stone_detector.py`.

## Статус детектора (на примерах)

| сцена | найдено | прим. |
|---|---|---|
| 1-stone | 1 | ок |
| 2-stones | 2 | ок |
| lots-of-stones | 6 | вкл. большой песчаный холм (добываемый) |
| stones-between-trees | 4 | кластер среди деревьев |
| full-map (город) | ~50 ложных | НЕкритично — ресурсы в природе, город панится мимо |
| coal-1 / coal-2 / coal-3 | 3 / 1 / 2 | уголь ловится (V≥40) |

Город шумит: серый булыжник по цвету = серая крыша/брусчатка/грунт, не
разделяются. Камни живут в природе (трава/деревья отсеяны цветом → там чисто),
поэтому город не давим. Ложные в городе отсеются «чёрным списком» при клике
(нет глоу добычи) — как у дерева.

Известный нюанс: песок и валун на холме = ОДИН цвет (V~88 оба), цветом не
разделить → холм ловим целиком как одну каменную формацию, кликаем центр.

## Сигнал добычи (замерено в живой игре)

У камня **НЕТ зелёного глоу** как у дерева (`diag_glow` показал neon≈0 у камня,
а яркая зелень ~4000px — фон/трава). Сигнал ИНВЕРТИРОВАН: перс приходит, ВСТАЁТ
на камень и заслоняет его телом → пиксели камня в зоне ПАДАЮТ; добыл и отошёл →
ВЕРНУЛИСЬ. Замер `diag_capture.py`: base≈1157 → упало до ~324 (добывает) →
вернулось 1157 (добыто).

`bot.py` ждёт по `stone_px`:
- `px <= base*ENGAGE_DROP_FRAC` (0.70) → перс взялся, добывает.
- `px >= base*RECOVER_FRAC` (0.90) → добыто.
- не упало за бюджет → недостижимо/ложное (строение) → чёрный список.

Инструмент калибровки: `python diag_capture.py` (кликает камень, логирует
`камень_px` + сохраняет кадры `cap_*.png`).

⚠️ Нюанс мультиплеера: чужой игрок, прошедший через зону, тоже роняет `px` →
ложный «добывает/добыто». v1-ограничение; шум гасится `VANISH_CONFIRM` и
`HARD_CAP=8` (максимум между добычами).

## Запуск бота (из папки stone-coal/)

1. Выставь `MONITOR_INDEX` в `bot.py` (твой монитор с игрой).
2. Сухой прогон: `DRY_RUN=True` → `python bot.py` (печать без кликов).
3. Бой: `DRY_RUN=False` → `python bot.py`. Стоп — **q**. Следи за логом
   `камень_px base/min/last` — по нему подкрути `ENGAGE_DROP_FRAC`/`RECOVER_FRAC`.

## Осталось / нюансы

- Сигнал добычи — v1-эвристика (заслон телом). Проверить в бою на потоке камней;
  если мажет — крутить `ENGAGE_DROP_FRAC`/`RECOVER_FRAC`/`STONE_GLOW_R`.
- Ложные в городе (ступени дома и пр.) отсекаются чёрным списком (перс не
  «возьмётся» → no_highlight).
- `CHAR_ANCHOR`, `MONITOR_INDEX`, пан — те же что у дерева (один сетап).
